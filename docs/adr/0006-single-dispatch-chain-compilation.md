# Edit chain compiles to a single RuntimeEffect per chain-config; chain-config cache keyed on ordered type list

The v1 editor rendered the **edit chain** as N nested `RuntimeShader` nodes, one per active **adjustment layer**, each preserving its own sRGB<->sRGB contract. The new model is F: a single `RuntimeEffect` whose source is generated by concatenating the active layer bodies, evaluated in one GPU pass, with a chain-config cache so reorders don't recompile every time.

## Why we left nested per-layer shaders

The nested model is simple but pays a real cost on every active layer: each per-layer `RuntimeShader` is a fullscreen GPU pass with a texture read, a texture write, and (in linear) a sRGB<->linear round-trip. On 5 active layers, that's 5 texture round-trips and 5 sRGB conversions. On a 24 MP source that's enough to drop slider FPS during a drag.

F collapses the N passes into one: one source read, one final write, one sRGB round-trip for the whole chain regardless of layer count. The chain operates entirely in linear light, with the body templates written against a `half3 color` variable that each layer transforms in turn. Slider drags only touch uniforms; chain composition changes (add/remove/reorder/visibility-toggle) invalidate the chain-config cache and trigger a regen.

Reference: [Film-Simulator `ImageProcessingShader.kt`](https://github.com/YahiaAngelo/Film-Simulator) does the same in a single static shader; we do it in a generated one because the chain composition is user-editable.

## What changed

### Per-layer shape

`LayerEntry.effect: SkRuntimeEffect` is replaced with `LayerEntry.body: BodyRenderer`. A `BodyRenderer` is `(layerIndex: number) => string` — it returns the SkSL statements that operate on the chain's shared `half3 color` variable, with uniforms namespaced by the layer's index in the chain (`l0_stops`, `l1_amount`, ...). The per-layer `shader.ts` files are gone; bodies live in `src/layers/bodies/<layer>.ts`.

### Chain source generator

`src/layers/chain-source.ts` walks the active layers, emits one `uniform half l{i}_{field};` per field, and concatenates the bodies into a single `main()` that linearizes the source, runs the bodies in order, clamps, and delinearizes. The sRGB<->linear helpers are prepended once.

### Chain-config cache

`src/layers/chain-cache.ts` is a `Map<chainKey, { source, effect }>`. Key is the ordered type list (`"exposure|contrast|grain"`). Slider drags don't touch the map; add/remove/reorder/visibility-toggle produce a new key and trigger a regen. Skia does not cache `RuntimeEffect` internally — the cache avoids re-parsing on every reorder.

### Pipeline shape

`<Image image={src}>{filter1}{filter2}...</Image>` (N nested shaders, N+1 texture passes) becomes `<Fill><Shader source={chainEffect} uniforms={...}><ImageShader image={src} fit="contain" rect={...}/></Shader></Fill>` (1 shader, 1 texture pass). `ImageShader`'s `fit` prop handles aspect-ratio containment that the old `<Image fit="contain">` did.

### Uniforms binding

The pipeline builds a single `useDerivedValue` of `Record<string, number>` that iterates the active layers and emits `l{i}_{field}` → `sv[field].value`. One worklet, one uniform map. Namespacing means two layers of the same type never collide; a future "allow duplicate layers" feature gets it for free.

## Tradeoffs accepted

- **CA and real clarity are deferred.** Both need direct `image.eval()` access (CA samples the source at offsets; real clarity needs a multi-tap local mean). In F the body operates on the accumulated `color`, not the source. The CA body emits a no-op for now; clarity is a ported midtone lift. Both are marked `// deferred` and called out in their body files.
- **Chain composition change pays a regen.** `Skia.RuntimeEffect.Make` re-parses the source on the JS thread (~5-30 ms on a phone). Dragging a slider does NOT pay this — the key is the type list, not the values. Reorders and visibility toggles do.
- **SkSL complexity limit is a soft ceiling.** A 9-layer chain is well under any GPU's per-shader ALU budget. Adding LUT (trilinear) and halation (multi-tap blur) might push it; we'll see frame drops before we hit hard compile errors.
- **Vignette aspect ratio is still wrong.** The body uses `coord * 2 - 1` (port of the old behavior), which produces an elliptical vignette on non-square canvases. The aspect-correct fix needs a `resolution` uniform; deferred.

## The contract

- **Each layer body**: `(layerIndex: number) => string`. Operates on `half3 color` in linear light. Local variables go in `{}` blocks.
- **Each layer's uniforms**: `uniform half l{index}_{field};` for every field in the registry entry. Slider values are `half`; we don't need `float` precision for the 0..N slider ranges.
- **Chain source**: one `uniform shader image;` plus the union of all layer uniforms, then `srgbToLinear` / `linearToSrgb` helpers, then `main()` with bodies inlined.
- **Cache key**: ordered type sequence. `invalidate()` exists for the dev workflow where a body template changes; not exposed in production paths.
